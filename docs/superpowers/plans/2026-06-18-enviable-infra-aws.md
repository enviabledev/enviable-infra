# enviable-infra AWS Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Provision, with Terraform, the AWS `eu-north-1` infrastructure that hosts the enviable-system NestJS backend (EC2 + Redis + Chromium, RDS Postgres, S3), front it with Caddy auto-TLS at `api.enviabletricycle.com`, and ship a GitHub Actions CI/CD pipeline that deploys the backend.

**Architecture:** A single t3.micro EC2 instance in a small dedicated VPC runs Caddy → NestJS → Redis via Docker Compose. Caddy terminates TLS (Let's Encrypt). Postgres is managed RDS db.t3.micro in private subnets; S3 holds PDFs/uploads. Secrets live in SSM Parameter Store. GitHub Actions authenticates via OIDC (no static keys): it builds the backend image to ECR, updates an image-tag SSM parameter, uploads the compose/Caddyfile to S3, then triggers an SSM Run Command on the box to pull and restart. The Next.js frontend stays on Vercel and only needs `BACKEND_API_URL`.

**Tech Stack:** Terraform ≥ 1.6, AWS provider ~> 5.0, Amazon Linux 2023, Docker + Compose v2, Caddy 2, Redis 7, Node 22 + Chromium, PostgreSQL 16, GitHub Actions + OIDC.

## Global Constraints

- Region: `eu-north-1` (Stockholm). All resources here.
- Compute: exactly **one** EC2 **t3.micro**, AL2023, gp3 root, **2 GB swapfile**.
- Database: **RDS PostgreSQL 16, db.t3.micro**, 20 GB gp3, **single-AZ**, `publicly_accessible = false`, 7-day backups, `storage_encrypted = true`, `deletion_protection = true`.
- Redis runs **on the EC2 box** as a container (not ElastiCache).
- TLS via **Caddy on the box** (Let's Encrypt). **No ALB. No NAT gateway. No Route 53 zone.**
- DNS stays at the **current provider**; we only output values for two new subdomain records (`api` A, `portal` CNAME). The zone is never migrated; apex/MX/DKIM untouched.
- Secrets in **SSM Parameter Store SecureString**; DB password + `SESSION_SECRET` generated by Terraform. Nothing secret committed.
- Shell access via **SSM Session Manager only** — no inbound SSH (port 22 never opened).
- CI/CD: **GitHub Actions + OIDC**, no long-lived AWS keys.
- Migrations: `prisma migrate deploy` runs on backend container start.
- **Repo guard:** in `enviable-system` and `enviable-web`, only **append new files**. Never edit or move an existing file there without explicit user permission. (See Task 16 checkpoint.)
- All Terraform must pass `terraform fmt -check` and `terraform validate` before commit.

## Task 2: Remote-state bootstrap (S3 bucket + DynamoDB lock)

This is a separate root with **local** state, applied once, to create the bucket/table the main stack's S3 backend uses.

**Files:**
- Create: `enviable-infra/bootstrap/main.tf`
- Create: `enviable-infra/bootstrap/variables.tf`
- Create: `enviable-infra/bootstrap/outputs.tf`

**Interfaces:**
- Produces: an S3 bucket `enviable-tfstate-<account_id>` and DynamoDB table `enviable-tf-lock`, referenced by `backend.tf` in Task 3.

- [ ] **Step 1: Write `bootstrap/variables.tf`**

```hcl
variable "region" {
  type    = string
  default = "eu-north-1"
}

variable "project" {
  type    = string
  default = "enviable"
}
```

- [ ] **Step 2: Write `bootstrap/main.tf`**

```hcl
terraform {
  required_version = ">= 1.6.0"
  required_providers {
    aws = { source = "hashicorp/aws", version = "~> 5.0" }
  }
}

provider "aws" {
  region = var.region
  default_tags {
    tags = { Project = var.project, ManagedBy = "terraform" }
  }
}

data "aws_caller_identity" "current" {}

resource "aws_s3_bucket" "tfstate" {
  bucket = "${var.project}-tfstate-${data.aws_caller_identity.current.account_id}"
}

resource "aws_s3_bucket_versioning" "tfstate" {
  bucket = aws_s3_bucket.tfstate.id
  versioning_configuration { status = "Enabled" }
}

resource "aws_s3_bucket_server_side_encryption_configuration" "tfstate" {
  bucket = aws_s3_bucket.tfstate.id
  rule {
    apply_server_side_encryption_by_default { sse_algorithm = "AES256" }
  }
}

resource "aws_s3_bucket_public_access_block" "tfstate" {
  bucket                  = aws_s3_bucket.tfstate.id
  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}

resource "aws_dynamodb_table" "tf_lock" {
  name         = "${var.project}-tf-lock"
  billing_mode = "PAY_PER_REQUEST"
  hash_key     = "LockID"
  attribute {
    name = "LockID"
    type = "S"
  }
}
```

- [ ] **Step 3: Write `bootstrap/outputs.tf`**

```hcl
output "state_bucket" { value = aws_s3_bucket.tfstate.bucket }
output "lock_table"   { value = aws_dynamodb_table.tf_lock.name }
```

- [ ] **Step 4: Apply the bootstrap** (requires AWS credentials configured for eu-north-1)

```bash

terraform apply
```
Expected: creates the S3 bucket and DynamoDB table. Note the `state_bucket` output value.

## Task 7: SSM secrets module (generated secrets + parameters)

**Files:**
- Create: `enviable-infra/modules/ssm/variables.tf`
- Create: `enviable-infra/modules/ssm/main.tf`
- Create: `enviable-infra/modules/ssm/outputs.tf`

**Interfaces:**
- Consumes: `project`, `default_initial_password`, `database_url`, `redis_url`, `invoice_config`, `s3_bucket`, `region`.
- Produces: `parameter_path_prefix` (= `/enviable/prod`), `session_secret` (sensitive), and the set of SSM parameters under that prefix. The deploy reads every param at the prefix with `--with-decryption`.

- [ ] **Step 1: Write `modules/ssm/variables.tf`**

```hcl
variable "project"                  { type = string }
variable "default_initial_password" { type = string, sensitive = true }
variable "database_url"             { type = string, sensitive = true }
variable "redis_url"                { type = string }
variable "s3_bucket"                { type = string }
variable "region"                   { type = string }
variable "invoice_config"           { type = map(string) }
```

- [ ] **Step 2: Write `modules/ssm/main.tf`**

```hcl
locals {
  prefix = "/${var.project}/prod"

  # Non-secret app config written as plain String params.
  plain_params = merge(
    {
      REDIS_URL                  = var.redis_url
      SESSION_STORE              = "redis"
      NODE_ENV                   = "production"
      PORT                       = "3000"
      PUPPETEER_EXECUTABLE_PATH  = "/usr/bin/chromium"
      AWS_REGION                 = var.region
      S3_BUCKET                  = var.s3_bucket
    },
    var.invoice_config,
  )
}

resource "random_password" "session_secret" {
  length  = 64
  special = false
}

resource "aws_ssm_parameter" "session_secret" {
  name  = "${local.prefix}/SESSION_SECRET"
  type  = "SecureString"
  value = random_password.session_secret.result
}

resource "aws_ssm_parameter" "default_initial_password" {
  name  = "${local.prefix}/DEFAULT_INITIAL_PASSWORD"
  type  = "SecureString"
  value = var.default_initial_password
}

resource "aws_ssm_parameter" "database_url" {
  name  = "${local.prefix}/DATABASE_URL"
  type  = "SecureString"
  value = var.database_url
}

resource "aws_ssm_parameter" "plain" {
  for_each = local.plain_params
  name     = "${local.prefix}/${each.key}"
  type     = "String"
  value    = each.value
}

# Image tag updated by CI on every deploy; seeded here so the param exists.
resource "aws_ssm_parameter" "backend_image" {
  name  = "${local.prefix}/BACKEND_IMAGE"
  type  = "String"
  value = "PENDING_FIRST_DEPLOY"

  lifecycle {
    ignore_changes = [value] # CI owns this value after first deploy
  }
}
```


## Task 17: Backend deploy workflow (NEW file in enviable-system)

**Files (append-only, new file):**
- Create: `enviable-system/.github/workflows/deploy.yml`

**Interfaces:**
- Consumes: GitHub repo secrets/vars `AWS_REGION`, `AWS_DEPLOY_ROLE_ARN` (= `backend_deploy_role_arn` output), `ECR_REPOSITORY_URL`, `S3_BUCKET`, `SSM_PREFIX`, `INSTANCE_ID`.
- Produces: a build→push→deploy pipeline on push to `main`.

- [ ] **Step 1: Write `enviable-system/.github/workflows/deploy.yml`**

```yaml
name: deploy-backend

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  id-token: write
  contents: read

env:
  AWS_REGION: ${{ vars.AWS_REGION }}
  ECR: ${{ vars.ECR_REPOSITORY_URL }}
  BUCKET: ${{ vars.S3_BUCKET }}
  SSM_PREFIX: ${{ vars.SSM_PREFIX }}
  INSTANCE_ID: ${{ vars.INSTANCE_ID }}

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS credentials (OIDC)
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ vars.AWS_DEPLOY_ROLE_ARN }}
          aws-region: ${{ env.AWS_REGION }}

      - name: Login to ECR
        run: aws ecr get-login-password --region "$AWS_REGION" | docker login --username AWS --password-stdin "${ECR%/*}"

      - name: Build and push image
        run: |
          IMAGE="$ECR:${GITHUB_SHA}"
          docker build -t "$IMAGE" .
          docker push "$IMAGE"
          echo "IMAGE=$IMAGE" >> "$GITHUB_ENV"

      - name: Record image tag in SSM
        run: aws ssm put-parameter --region "$AWS_REGION" --name "$SSM_PREFIX/BACKEND_IMAGE" --type String --overwrite --value "$IMAGE"

      - name: Upload deploy artifacts to S3
        run: aws s3 cp deploy/ "s3://$BUCKET/_deploy/" --recursive

      - name: Trigger deploy on instance (SSM Run Command)
        run: |
          CMD_ID=$(aws ssm send-command \
            --region "$AWS_REGION" \
            --instance-ids "$INSTANCE_ID" \
            --document-name "AWS-RunShellScript" \
            --comment "deploy ${GITHUB_SHA}" \
            --parameters commands='[
              "set -euo pipefail",
              "cd /opt/enviable",
              "aws s3 sync s3://'"$BUCKET"'/_deploy/ /opt/enviable/",
              "umask 077",
              ": > /opt/enviable/.env",
              "for p in $(aws ssm get-parameters-by-path --path '"$SSM_PREFIX"' --with-decryption --recursive --query \"Parameters[].Name\" --output text); do k=$(basename \"$p\"); v=$(aws ssm get-parameter --name \"$p\" --with-decryption --query Parameter.Value --output text); echo \"$k=$v\" >> /opt/enviable/.env; done",
              "aws ecr get-login-password --region '"$AWS_REGION"' | docker login --username AWS --password-stdin '"${ECR%/*}"'",
              "docker compose -f /opt/enviable/docker-compose.yml pull",
              "docker compose -f /opt/enviable/docker-compose.yml up -d",
              "docker image prune -f"
            ]' \
            --query "Command.CommandId" --output text)
          echo "Command: $CMD_ID"
          aws ssm wait command-executed --region "$AWS_REGION" --command-id "$CMD_ID" --instance-id "$INSTANCE_ID" || true
          aws ssm get-command-invocation --region "$AWS_REGION" --command-id "$CMD_ID" --instance-id "$INSTANCE_ID" --query "{Status:Status,Out:StandardOutputContent,Err:StandardErrorContent}" --output json
```

```bash
git commit -m "ci: deploy backend to AWS via ECR + SSM (OIDC)"
```
